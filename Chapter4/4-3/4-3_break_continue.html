4-3. break / continue

難易度：⭐⭐⭐☆☆　|　問題数：3問　|　形式：出力を答える・コード記述
📖 解説

break文とは？

ループを即座に終了させる命令です。switch 文でも使いましたが、for や while でも同様に機能します。

break の例（i == 5 でループ終了）
for (int i = 0; i < 10; i++) {
    if (i == 5) {
        break;  // i が 5 になったらループを終了
    }
    System.out.println(i);
}
// → 0  1  2  3  4

continue文とは？

その回の処理だけをスキップして次の繰り返しに進む命令です。ループ自体は終了しません。

continue の例（偶数をスキップ）
for (int i = 0; i < 10; i++) {
    if (i % 2 == 0) {
        continue;  // 偶数のときはスキップ
    }
    System.out.println(i);
}
// → 1  3  5  7  9

命令動作ループへの影響
breakその時点でループを完全に終了ループを抜ける
continueその回の処理だけスキップ次の繰り返しに進む

while(true) と break の組み合わせ

while (true) で意図的に無限ループを作り、条件を満たしたときに break で抜けるパターンがよく使われます。

while(true) + break の例
int i = 0;

while (true) {      // 無限ループ
    if (i >= 5) {
        break;      // 条件を満たしたら脱出
    }
    System.out.println(i);
    i++;
}
System.out.println("終了");
// → 0  1  2  3  4  終了

ネストしたループでの break

break や continue は自分が属する最も内側のループにのみ作用します。

ネストの break（内側のループのみ終了）
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (j == 1) {
            break;  // 内側のループだけ終了
        }
        System.out.println(i + "-" + j);
    }
}
// → 0-0  1-0  2-0
💡 外側のループを終了させたい場合
ラベル付き break（outer: for(...) → break outer;）で外側のループを終了できますが、読みにくくなります。
代わりに フラグ変数 を使うか、別メソッドに切り出す方がすっきりします。

コード例：素数判定

PrimeChecker.java
public class PrimeChecker {
    public static void main(String[] args) {
        int num = 17;
        boolean isPrime = true;

        if (num < 2) {
            isPrime = false;
        } else {
            for (int i = 2; i < num; i++) {
                if (num % i == 0) {
                    isPrime = false;
                    break;  // 割り切れた時点で終了（以降は調べる必要なし）
                }
            }
        }

        System.out.println(num + " は素数: " + isPrime);  // 17 は素数: true
    }
}

問題 4-3-2
出力を答える ⭐⭐☆

次のコードを実行したとき、出力される結果を答えてください。

Q2.java
public class Q2 {
    public static void main(String[] args) {
        int count = 0;

        for (int i = 1; i <= 20; i++) {
            if (i % 2 == 0) continue;
            if (i % 5 == 0) break;
            count++;
            System.out.print(i + " ");
        }

        System.out.println();
        System.out.println("count: " + count);
    }
}
▶ 答えを見る
▶ 出力結果
1 3 
count: 2
解説
  • i = 1：奇数・5の倍数でない → 出力。count = 1
  • i = 2：偶数 → continue（スキップ）
  • i = 3：奇数・5の倍数でない → 出力。count = 2
  • i = 4：偶数 → continue（スキップ）
  • i = 5：奇数 → 偶数チェックはパス、i % 5 == 0 → break（終了）
  • 5 は break で終了するため出力されず、count もカウントされません。
問題 4-3-3
コード記述 ⭐⭐⭐

次の条件を満たす NumberFilter クラスを作成してください。

  • for 文で 1 から 30 まで繰り返す
  • 4の倍数のときは continue（スキップ）する
  • 出力した数の合計が 100 を超えたら break（ループ終了）する
  • スキップ・終了した数以外を出力し、最後に合計を出力する
▶ 期待される出力
1 2 3 5 6 7 9 10 11 13 14 15 17 18 19 21 
合計: 171
💡 ヒント：continue の前に合計を加算しないように注意しましょう。
合計チェック（sum > 100）は加算・出力の後に行います。
▶ 答えを見る
NumberFilter.java
public class NumberFilter {
    public static void main(String[] args) {
        int sum = 0;

        for (int i = 1; i <= 30; i++) {
            if (i % 4 == 0) {
                continue;  // 4の倍数はスキップ
            }
            sum += i;
            System.out.print(i + " ");
            if (sum > 100) {
                break;  // 合計が100を超えたら終了
            }
        }

        System.out.println();
        System.out.println("合計: " + sum);
    }
}
解説
  • 4の倍数（4・8・12・16・20・24・28）はスキップされます。
  • それ以外の数を順に加算し、sum > 100 になった時点で break。
  • 21 を加算した時点で合計が 171 となり100を超えるため終了。
  • break は加算・出力の後に評価されるため、21 は出力されます。
