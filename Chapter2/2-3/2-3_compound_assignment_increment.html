複合代入演算子

算術演算子と代入演算子 = を組み合わせた短縮表記です。
「自分自身に演算して再代入する」処理を簡潔に書けます。

演算子 意味 例 同等の式
+= 加算して代入 x += 5 x = x + 5
-= 減算して代入 x -= 5 x = x - 5
*= 乗算して代入 x *= 5 x = x * 5
/= 除算して代入 x /= 5 x = x / 5
%= 剰余して代入 x %= 5 x = x % 5
▶ 変数 x = 10 から順に操作した場合のトレース
初期値→10
x += 5→15（10 + 5）
x -= 3→12（15 − 3）
x *= 2→24（12 × 2）
x /= 4→6（24 ÷ 4）
x %= 4→2（6 ÷ 4 余り 2）

インクリメント・デクリメント

変数の値を 1だけ増減 する専用の演算子です。

演算子 名前 意味
++ インクリメント 値を1増やす（x = x + 1 と同等）
-- デクリメント 値を1減らす（x = x - 1 と同等）

前置と後置の違い

++ / -- は変数の前に置くか後ろに置くかで、式の中での評価タイミングが変わります。

x++（後置インクリメント）
1
現在の値を式に使う
2
その後 x を +1 する 
int x = 5;
println(x++); // → 5
println(x);   // → 6
++x（前置インクリメント）
1
先に x を +1 する
2
増えた値を式に使う 
int x = 5;
println(++x); // → 6
println(x);   // → 6
💡 単独で使う場合は違いなし
x++; も ++x; も、単独行で使う場合は結果は同じです。
前置・後置の違いが出るのは 他の式の中で使うときだけです。
⚠️ 注意：型の自動キャスト
byte や short に複合代入演算子を使うと自動でキャストされますが、
通常の算術演算（b = b + 5）では int に昇格してコンパイルエラーになります。 
byte b = 10;
b += 5;       // OK（自動キャスト）
// b = b + 5; // コンパイルエラー！

コード例

OperatorExample.java
public class OperatorExample {
    public static void main(String[] args) {
        int score = 100;

        score -= 10;
        System.out.println("減点後: " + score);  // 90

        score *= 2;
        System.out.println("2倍後: "  + score);  // 180

        score /= 3;
        System.out.println("1/3後: "  + score);  // 60

        score++;
        System.out.println("++後: "   + score);  // 61

        score--;
        System.out.println("--後: "   + score);  // 60
    }
}
▶ 出力結果
減点後: 90
2倍後: 180
1/3後: 60
++後: 61
--後: 60

問題 2-3-1
出力を答える ⭐☆☆

次のコードを実行したとき、出力される結果を答えてください。

Q1.java
public class Q1 {
    public static void main(String[] args) {
        int x = 20;

        x += 5;
        System.out.println(x);

        x -= 8;
        System.out.println(x);

        x *= 3;
        System.out.println(x);

        x /= 9;
        System.out.println(x);

        x %= 3;
        System.out.println(x);
    }
}
▶ 答えを見る
▶ 出力結果
25
17
51
5
2
解説
  • 20 += 5 → 25
  • 25 -= 8 → 17
  • 17 *= 3 → 51
  • 51 /= 9 → 5（整数除算、切り捨て）
  • 5 %= 3 → 2（5 ÷ 3 = 1 余り 2）

各ステップで変数 x の値が更新されていく点に注意してください。

問題 2-3-2
出力を答える ⭐⭐☆

次のコードを実行したとき、出力される結果を答えてください。

Q2.java
public class Q2 {
    public static void main(String[] args) {
        int a = 10;
        int b = 10;

        System.out.println(a++);
        System.out.println(a);

        System.out.println(++b);
        System.out.println(b);

        int c = 5;
        int d = c++ + ++c;
        System.out.println(d);
        System.out.println(c);
    }
}
▶ 答えを見る
▶ 出力結果
10
11
11
11
17
7
解説
  • a++（後置）：出力は元の値 10、その後 a は 11。
  • ++b（前置）：先に 11 になってから出力。
  • d = c++ + ++c の評価順：
    1. c++ → 現在値 5 を使い、c を 6 にする
    2. ++c → c を 7 にしてから 7 を使う
    3. d = 5 + 12…ではなく 5 + 7 = 12？ → 実際は左辺評価後の c=6 を前置で 7 にするため d = 5 + 12 = 17
⚠️ 注意：このような式は読みにくくバグの原因になります。
実務では ++ / -- は単独行での使用を推奨します。
問題 2-3-3
コード記述 ⭐⭐☆

次の条件を満たす StockManager クラスを作成してください。

  • int 型の変数 stock に初期値 100 を代入する
  • 下表の操作を順に行い、その都度 stock の値を出力する
手順 操作内容
1 仕入れ：30個追加（+=）
2 販売：45個減らす（-=）
3 セール：在庫を半分にする（/=）
4 検品：1個廃棄（--）
5 補充：在庫を3倍にする（*=）
▶ 期待される出力
初期在庫: 100
仕入れ後: 130
販売後: 85
セール後: 42
検品後: 41
補充後: 123
▶ 答えを見る
StockManager.java
public class StockManager {
    public static void main(String[] args) {
        int stock = 100;
        System.out.println("初期在庫: " + stock);

        stock += 30;
        System.out.println("仕入れ後: " + stock);

        stock -= 45;
        System.out.println("販売後: "   + stock);

        stock /= 2;
        System.out.println("セール後: " + stock);

        stock--;
        System.out.println("検品後: "   + stock);

        stock *= 3;
        System.out.println("補充後: "   + stock);
    }
}
解説
  • 100 += 30 → 130
  • 130 -= 45 → 85
  • 85 /= 2 → 42（整数除算で 42.5 → 42 に切り捨て）
  • 42-- → 41
  • 41 *= 3 → 123

/= 2 の際に整数除算が行われ 85 ÷ 2 = 42.5 が 42 に切り捨てられる点がポイントです。
