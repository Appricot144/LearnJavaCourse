ポリモーフィズムとは？

同じ操作（メソッド呼び出し）に対して、オブジェクトの種類によって異なる振る舞いをする性質です。

種類 仕組み 解決タイミング
コンパイル時 メソッドのオーバーロード コンパイル時
実行時 メソッドのオーバーライド＋継承/インターフェース 実行時（動的ディスパッチ）

オーバーライドとは？

サブクラスが親クラスのメソッドを再定義することです。@Override アノテーションで意図を明示します。

▶ Animal を継承した Dog / Cat / Bird
Animal　speak() → "..."
↓ extends
Dog
speak() → "ワン！"
Cat
speak() → "ニャー！"
Bird
speak() → "チュン！"
オーバーライドの例
class Animal {
    public void speak() { System.out.println("..."); }
}

class Dog extends Animal {
    @Override
    public void speak() { System.out.println("ワン！"); }
}

class Cat extends Animal {
    @Override
    public void speak() { System.out.println("ニャー！"); }
}

参照型の多態性（動的ディスパッチ）

親クラス型の変数でサブクラスのインスタンスを参照できます。
呼ばれるメソッドは実際のオブジェクトの型によって決まります。

▶ Animal 型でも、実際のオブジェクトの speak() が呼ばれる
Animal a1 = new Dog()
→
Dog の speak()
→ "ワン！"
Animal a2 = new Cat()
→
Cat の speak()
→ "ニャー！"
配列とポリモーフィズム
Animal[] animals = { new Dog(), new Cat(), new Dog() };

for (Animal a : animals) {
    a.speak();  // それぞれの実装が呼ばれる
}
// → ワン！
// → ニャー！
// → ワン！

instanceof 演算子

型チェックとダウンキャスト
Animal a = new Dog();

System.out.println(a instanceof Animal);  // true
System.out.println(a instanceof Dog);     // true
System.out.println(a instanceof Cat);     // false

// instanceof でチェックしてからダウンキャスト
if (a instanceof Dog) {
    Dog dog = (Dog) a;   // ダウンキャスト
    dog.fetch();         // Dog 固有のメソッド
}

toString() のオーバーライド

println で自動的に呼ばれる toString()
class Point {
    int x, y;
    public Point(int x, int y) { this.x = x; this.y = y; }

    @Override
    public String toString() {
        return "Point(" + x + ", " + y + ")";
    }
}

Point p = new Point(3, 4);
System.out.println(p);           // Point(3, 4)
System.out.println("座標: " + p); // 座標: Point(3, 4)

コード例：図形の面積計算

Shape / Circle / Rectangle
class Shape {
    String color;
    public Shape(String color) { this.color = color; }
    public double area() { return 0.0; }
    public void show() {
        System.out.printf("%s（%s）面積: %.2f%n",
            getClass().getSimpleName(), color, area());
    }
}

class Circle extends Shape {
    double radius;
    public Circle(String color, double radius) {
        super(color); this.radius = radius;
    }
    @Override
    public double area() { return Math.PI * radius * radius; }
}

class Rectangle extends Shape {
    double width, height;
    public Rectangle(String color, double width, double height) {
        super(color); this.width = width; this.height = height;
    }
    @Override
    public double area() { return width * height; }
}

// Main
Shape[] shapes = {
    new Circle("赤", 5.0),
    new Rectangle("青", 4.0, 6.0),
    new Circle("緑", 3.0)
};

double total = 0;
for (Shape s : shapes) { s.show(); total += s.area(); }
System.out.printf("合計面積: %.2f%n", total);
▶ 出力結果
Circle（赤）面積: 78.54
Rectangle（青）面積: 24.00
Circle（緑）面積: 28.27
合計面積: 130.81

問題 8-2-1
出力を答える ⭐⭐☆

次のコードを実行したとき、出力される結果を答えてください。

Q1.java
class Vehicle {
    String name;
    public Vehicle(String name) { this.name = name; }
    public String move() { return name + " が移動する"; }

    @Override
    public String toString() {
        return "[" + getClass().getSimpleName() + "] " + name;
    }
}

class Car   extends Vehicle {
    public Car(String name)   { super(name); }
    @Override public String move() { return name + " が走る 🚗"; }
}
class Boat  extends Vehicle {
    public Boat(String name)  { super(name); }
    @Override public String move() { return name + " が泳ぐ 🚢"; }
}
class Plane extends Vehicle {
    public Plane(String name) { super(name); }
    @Override public String move() { return name + " が飛ぶ ✈"; }
}

public class Q1 {
    public static void main(String[] args) {
        Vehicle[] fleet = {
            new Car("トヨタ"), new Boat("フェリー"),
            new Plane("JAL"),  new Car("ホンダ")
        };

        for (Vehicle v : fleet) System.out.println(v.move());
        System.out.println("---");
        for (Vehicle v : fleet) {
            System.out.println(v);
            System.out.println(v instanceof Car);
        }
    }
}
▶ 答えを見る
▶ 出力結果
トヨタ が走る 🚗
フェリー が泳ぐ 🚢
JAL が飛ぶ ✈
ホンダ が走る 🚗
---
[Car] トヨタ
true
[Boat] フェリー
false
[Plane] JAL
false
[Car] ホンダ
true
解説
  • Vehicle[] に混在させ、move() を呼ぶと各クラスのオーバーライドが実行されます。
  • toString() は getClass().getSimpleName() でサブクラス名を取得します。
  • v instanceof Car は Car のインスタンスのみ true になります。
問題 8-2-2
出力を答える ⭐⭐⭐

Q2.java
class Animal {
    private String name;
    public Animal(String name) { this.name = name; }
    public String getName()    { return name; }
    public String speak()      { return "..."; }
    public String action()     { return speak() + " と言って動く"; }
}

class Dog extends Animal {
    public Dog(String name) { super(name); }
    @Override public String speak() { return "ワン"; }
}

class PoliceDog extends Dog {
    public PoliceDog(String name) { super(name); }
    @Override public String speak() { return "ガウ"; }
    public String guard() { return getName() + ": " + action(); }
}

public class Q2 {
    public static void main(String[] args) {
        Animal    a  = new Animal("動物");
        Dog       d  = new Dog("ポチ");
        PoliceDog pd = new PoliceDog("レックス");

        System.out.println(a.action());
        System.out.println(d.action());
        System.out.println(pd.action());
        System.out.println(pd.guard());

        Animal ref = pd;
        System.out.println(ref.speak());
        System.out.println(ref instanceof Dog);
        System.out.println(ref instanceof PoliceDog);
    }
}
▶ 答えを見る
▶ 出力結果
... と言って動く
ワン と言って動く
ガウ と言って動く
レックス: ガウ と言って動く
ガウ
true
true
解説
  • action() は内部で speak() を呼びます。動的ディスパッチにより実際のオブジェクトの speak() が呼ばれます。
  • pd.action() → PoliceDog の speak() = "ガウ" → "ガウ と言って動く"。
  • Animal ref = pd でも ref.speak() は PoliceDog の実装 "ガウ" が呼ばれます。
  • pd は PoliceDog → Dog → Animal の順に継承するため、instanceof Dog も instanceof PoliceDog も true。
問題 8-2-3
コード記述 ⭐⭐⭐

次のクラス群を作成してください。

クラス 内容
Employee name・baseSalary フィールド。calcSalary() は baseSalary を返す。show() で給与を出力
Manager Employee を継承。bonus フィールド追加。calcSalary() を baseSalary + bonus にオーバーライド
PartTimer Employee を継承。hoursWorked フィールド追加。calcSalary() を baseSalary × hoursWorked にオーバーライド
▶ 期待される出力
[田中] 給与: 300000円
[佐藤] 給与: 450000円
[山田] 給与: 112000円
---
合計給与: 862000円
💡 ヒント：Employee[] に3種を混在させて一括処理します。
show() 内の calcSalary() は動的ディスパッチで各クラスの実装が呼ばれます。
▶ 答えを見る
Employee / Manager / PartTimer + Main 
class Employee {
    String name;
    int    baseSalary;

    public Employee(String name, int baseSalary) {
        this.name       = name;
        this.baseSalary = baseSalary;
    }

    public int calcSalary() { return baseSalary; }

    public void show() {
        System.out.println("[" + name + "] 給与: " + calcSalary() + "円");
    }
}

class Manager extends Employee {
    int bonus;
    public Manager(String name, int baseSalary, int bonus) {
        super(name, baseSalary);
        this.bonus = bonus;
    }
    @Override
    public int calcSalary() { return baseSalary + bonus; }
}

class PartTimer extends Employee {
    int hoursWorked;
    public PartTimer(String name, int hourlyWage, int hoursWorked) {
        super(name, hourlyWage);
        this.hoursWorked = hoursWorked;
    }
    @Override
    public int calcSalary() { return baseSalary * hoursWorked; }
}

public class Main {
    public static void main(String[] args) {
        Employee[] staff = {
            new Employee("田中",  300000),
            new Manager("佐藤",  300000, 150000),
            new PartTimer("山田", 1400,   80)
        };

        int total = 0;
        for (Employee e : staff) {
            e.show();
            total += e.calcSalary();
        }
        System.out.println("---");
        System.out.println("合計給与: " + total + "円");
    }
}
解説
  • Employee[] に3種を混在させ、e.show() と e.calcSalary() を統一的に呼べます。
  • show() 内の calcSalary() は動的ディスパッチで各サブクラスの実装が呼ばれます。
  • Manager：300000 + 150000 = 450000。
  • PartTimer：1400 × 80 = 112000。
