JavaとJDKの全体像

これまでのコースで import java.util.* や import java.time.* を当然のように使ってきました。
これらは一体どこから来るのでしょうか？ まず Java を取り巻くソフトウェアの層を整理しましょう。

▶ Java アプリケーションの層構造
🧑‍💻 あなたのアプリケーション
Main.java、ProductService.java、…（本コースで書いたコード）
↓ 使う
📦 サードパーティライブラリ
Spring Boot、JUnit、Log4j、Jackson、Hibernate、…
Maven / Gradle でプロジェクトに追加する外部ライブラリ
↓ 使う
📚 Java 標準ライブラリ（JCL: Java Class Library）
java.lang、java.util、java.time、java.io、java.net、java.math、…
JDK に同梱。import するだけで使えるクラス群
↓ 含む
🔧 JDK（Java Development Kit）
javac（コンパイラ）、java（実行ツール）、jar、javadoc、… + JRE を含む
↓ 動く
⚙️ JVM（Java Virtual Machine）
バイトコードを実行する仮想マシン。Windows / macOS / Linux の違いを吸収する
「Write Once, Run Anywhere」を支える仕組み
💡 JDK と JRE の違い
JRE（Java Runtime Environment）はアプリを実行するだけの環境。
JDK（Java Development Kit）は開発に必要なコンパイラなどを含む。開発者は JDK をインストールします。
Java 11 以降はこの区分が統合され、JDK のみが配布されています。

標準ライブラリのパッケージ全体像

本コースで使ったパッケージはほんの一部です。JDK には 200 以上のパッケージが含まれています。
主要なものを「本コースで使ったか」という観点でマッピングしてみましょう。

java.lang
✅ 使用済み
String、Math、Integer、System、StringBuilder — Java の基盤
java.util
✅ 使用済み
List、Map、Set、Arrays、Collections、Optional
java.util.stream
✅ 使用済み
Stream、Collectors、IntStream
java.util.function
✅ 使用済み
Function、Predicate、Consumer、Supplier
java.time
✅ 使用済み
LocalDate、LocalDateTime、Period、Duration
java.time.format
✅ 使用済み
DateTimeFormatter
java.io
📖 未使用
ファイル入出力。File、Reader、Writer、InputStream
java.nio.file
📖 未使用
モダンなファイル操作。Path、Files、Paths
java.net
📖 未使用
ネットワーク通信。URL、HttpURLConnection、Socket
java.math
📖 未使用
高精度計算。BigDecimal（金額計算に必須）、BigInteger
java.lang.reflect
📖 未使用
リフレクション。クラス情報を実行時に取得・操作
java.util.concurrent
📖 未使用
並行処理。ExecutorService、Future、BlockingQueue
⚠️ java.math.BigDecimal に注意
金融・会計システムでは double の浮動小数点誤差が問題になります。
例えば 0.1 + 0.2 は Java で 0.30000000000000004 になります。
金額の計算には必ず BigDecimal を使いましょう。
3
GUIアプリ開発のライブラリ

Java でウィンドウ・ボタン・テキストボックスなどを持つデスクトップアプリを作るには
GUI ライブラリを使います。

🖼️ デスクトップ GUI ライブラリ
JavaFX
標準（別途追加） 現代的
Java 8〜10 は JDK に同梱、Java 11 以降はオープンソースプロジェクト（OpenJFX）として別途追加します。 FXML（XMLでUI設計）、CSS スタイリング、アニメーション、チャートなどモダンな機能を持つ現在の主流です。
// ボタンを持つウィンドウを作る例
Button btn = new Button("クリック");
btn.setOnAction(e -> System.out.println("押された！"));
Scene scene = new Scene(new StackPane(btn), 300, 200);
stage.setScene(scene);
stage.show();
Swing
JDK 標準（javax.swing）
JDK に同梱されており import するだけで使えます（import javax.swing.*;）。 Java 1.2 から存在する歴史あるライブラリで、今でも多くのレガシーシステムで使われています。 NetBeans IDE 自体も Swing で作られています。
// シンプルなウィンドウを作る（import 不要）
JFrame frame = new JFrame("Hello Swing");
JButton button = new JButton("クリック");
frame.add(button);
frame.setSize(300, 200);
frame.setVisible(true);
AWT（Abstract Window Toolkit）
JDK 標準（java.awt）
Java 最初期から存在する GUI ライブラリ（import java.awt.*;）。 Swing の土台となっているクラス群で、Color・Font・Graphics などは今も Swing / JavaFX で使われます。 単体で新規 GUI 開発に使うことはほぼありません。
4
Web開発のライブラリ・フレームワーク

Java は企業システムの Web バックエンドとして世界中で使われています。
HTTP リクエストを受け取り、データベースと連携してレスポンスを返す仕組みを提供します。

🌐 Web アプリケーション開発
Spring Boot
サードパーティ 業界標準・最人気
Java の Web 開発において圧倒的なシェアを持つフレームワーク。 数行のコードで REST API サーバが立ち上がります。依存性注入（DI）、自動設定、豊富なエコシステムが特徴。 Spring Framework を使いやすくラップしたものが Spring Boot です。
// これだけで GET /hello に応答する Web サーバが動く！
@RestController
public class HelloController {
    @GetMapping("/hello")
    public String hello() {
        return "Hello, World!";
    }
}
Jakarta EE（旧 Java EE）
エンタープライズ標準
Java のエンタープライズ向け標準仕様。Servlet、JSP、JPA、EJB などの仕様が含まれます。 WildFly や Payara などのアプリケーションサーバで動作します。 大規模な銀行・官公庁システムでよく使われます。
Micronaut / Quarkus
サードパーティ クラウドネイティブ
起動が高速でメモリ消費が少ないモダンなフレームワーク。 GraalVM を使ったネイティブコンパイルに対応し、クラウド・マイクロサービス向けに注目されています。 Spring Boot の代替として採用が増えています。
Servlet / JSP（javax.servlet）
Jakarta EE 標準
Java の Web の基礎。HTTP リクエスト・レスポンスを扱う低レベル API です。 Spring Boot などの上位フレームワークの土台になっています。 「Tomcat」というサーバでよく動かされます。
5
データベース連携のライブラリ
🗄️ データベース（DB）アクセス
JDBC（Java Database Connectivity）
JDK 標準（java.sql）
Java からデータベースに接続する標準 API（import java.sql.*;）。 MySQL・PostgreSQL・Oracle など、ドライバを追加すれば様々な DB と通信できます。 低レベルな API で、直接 SQL を書く場合に使います。
// SQL を直接実行する例
Connection conn = DriverManager.getConnection(url, user, pass);
PreparedStatement ps = conn.prepareStatement(
    "SELECT * FROM users WHERE id = ?");
ps.setInt(1, userId);
ResultSet rs = ps.executeQuery();
JPA / Hibernate
Jakarta EE 標準 / 実装 ORM
JPA（Jakarta Persistence API）は Java のオブジェクトとデータベースのテーブルをマッピングする標準仕様（ORM: Object-Relational Mapping）。 Hibernate はその代表的な実装です。SQL を書かずに Java オブジェクトとして DB を操作できます。
// アノテーションでテーブルと対応付ける
@Entity
@Table(name = "users")
public class User {
    @Id @GeneratedValue
    private Long id;
    private String name;
}
MyBatis
サードパーティ
SQL を XML やアノテーションで記述し、Java オブジェクトとマッピングするライブラリ。 JPA より SQL を細かく制御したい場合に向いています。日本の企業システムでも広く使われています。
6
テストライブラリ
🧪 自動テスト
JUnit 5
サードパーティ デファクトスタンダード
Java のユニットテストフレームワーク。@Test アノテーションを付けたメソッドが自動実行されます。 IDE（IntelliJ IDEA / Eclipse）に統合されており、コードが正しく動くかを自動検証できます。 プロの Java 開発ではほぼ必須のライブラリです。
// テストの例（assertEquals で期待値と実際値を比較）
@Test
void testAdd() {
    Calculator calc = new Calculator();
    assertEquals(5, calc.add(2, 3)); // 2+3=5 を検証
}
Mockito
サードパーティ
依存するクラスの「モック（偽物）」を作るライブラリ。 例えばデータベースに接続せずにサービス層だけをテストしたい場合に使います。 JUnit と組み合わせて使うのが一般的です。
7
ビルドツールと依存管理

サードパーティライブラリを使うには、それを「依存関係」としてプロジェクトに追加する必要があります。
ビルドツールがライブラリのダウンロード・コンパイル・テスト・パッケージングを自動化します。

🔨 ビルドツール
Maven
最もよく使われる
pom.xml ファイルでプロジェクト設定・依存関係を管理します。 「Maven Central」リポジトリからライブラリを自動ダウンロードします。 Java エンタープライズ開発の標準的な選択肢です。
<!-- pom.xml: Spring Boot を依存関係に追加する例 -->
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-web</artifactId>
  <version>3.2.0</version>
</dependency>
Gradle
Groovy / Kotlin DSL Android 標準
build.gradle（または build.gradle.kts）で設定を記述します。 Maven より柔軟で高速。Android アプリ開発では Gradle が標準的に使われます。 近年は Java/Kotlin プロジェクトでも採用が増えています。
// build.gradle.kts: JUnit を追加する例
dependencies {
    testImplementation("org.junit.jupiter:junit-jupiter:5.10.0")
}
8
その他の注目ライブラリ
🔭 知っておくと役立つライブラリ
Jackson / Gson
JSON 処理 Web API で必須
Java オブジェクトと JSON を相互変換（シリアライズ/デシリアライズ）するライブラリ。 Web API との通信では JSON が標準フォーマットなので、実務では必ず使います。 Jackson は Spring Boot に標準搭載されています。
// Java オブジェクト → JSON 文字列
ObjectMapper mapper = new ObjectMapper();
String json = mapper.writeValueAsString(product);
// → {"name":"りんご","price":150}
Lombok
コード生成 ボイラープレート削減
アノテーションを付けるだけで getter/setter/constructor/toString などを自動生成するライブラリ。 大量の定型コードを省略できるため、Java プロジェクトで広く使われています。 Java 16 以降の record に置き換えられつつある部分もあります。
// @Data でゲッター・セッター・toString などを全自動生成
@Data
@AllArgsConstructor
public class User {
    private String name;
    private int age;
}
// → getName()、setName()、toString() などが自動生成される
Log4j / SLF4J / Logback
ログ出力
本格的なログ出力ライブラリ。System.out.println の代わりにログレベル（DEBUG / INFO / WARN / ERROR）で分類して記録します。 SLF4J はログ API の標準インターフェース、Logback はその実装です。
Apache Commons
汎用ユーティリティ
Apache が提供する汎用ユーティリティ群。commons-lang3（文字列操作・反射）、 commons-collections4（コレクション拡張）、commons-io（ファイル操作）など多数のサブプロジェクトがあります。
9
JVM 上で動く他の言語

Java の JVM は Java 専用ではありません。JVM 上で動く様々な言語があり、
Java のライブラリをそのまま使いながら別の言語の特徴を活かせます。

🚀 JVM 言語
Kotlin
Java の後継候補 Android 公式言語
JetBrains 社が開発した JVM 言語。Java と 100% 互換で Java ライブラリがそのまま使えます。 Null 安全・データクラス・コルーチン（非同期処理）などが特徴で、Java より簡潔に書けます。 Google が Android の公式言語として採用しており、Spring Boot でも Kotlin が使えます。
// Kotlin で書くと Java より簡潔（同じ JVM で動く）
data class User(val name: String, val age: Int)
val user = User("Alice", 25)
println(user) // User(name=Alice, age=25)
Scala
関数型 + オブジェクト指向
関数型プログラミングとオブジェクト指向を融合した言語。Apache Spark（ビッグデータ処理）は Scala で書かれています。 Java より表現力が高い一方、学習コストも高めです。
Groovy
スクリプト言語
Java に動的型付けや Closure などを追加した言語。Gradle のビルドスクリプトの言語として有名です。 Java コードがほぼそのまま動くため、Java から Groovy への移行が容易です。
