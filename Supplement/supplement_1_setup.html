📦 セットアップの全体像

Java でプログラムを書いて動かすためには、まず JDK（Java Development Kit） をインストールする必要があります。JDK には Java のコンパイラ（javac）と実行環境（JVM）が含まれています。

▶ Java プログラムができるまでの流れ
✍️
コードを書く
Main.java
→
⚙️
コンパイル
javac Main.java
→
📄
バイトコード生成
Main.class
→
▶️
JVM で実行
java Main

インストールするものは JDK のみです。JDK をインストールすれば、コンパイラ（javac）も JVM（java）も一緒に手に入ります。

🔢 どのバージョン・ディストリビューションを選ぶか

JDK には「バージョン」と「ディストリビューション（提供元）」の2つの選択があります。

バージョンの選び方

バージョン 種別 おすすめ度 備考
Java 21 LTS ⭐⭐⭐⭐⭐ 推奨 2023年リリース。現時点の最新 LTS。本コースはこれで動作確認済み
Java 17 LTS ⭐⭐⭐⭐☆ 2021年リリース。企業システムで今も広く使われている安定版
Java 11 LTS ⭐⭐⭐☆☆ 2018年リリース。古いプロジェクトで使われることがある
Java 22 / 23 … 非LTS △ 学習目的なら可 最新機能が使えるが半年でサポート終了。業務では非推奨
💡 LTS（Long Term Support）とは
LTS 版は長期間のセキュリティアップデートが保証されるバージョンです。
学習・業務ともに LTS 版（特に Java 21）を選んでおくのが安全です。

ディストリビューションの選び方

同じ Java バージョンでも、配布元（ディストリビューション）が複数あります。どれも動作は同じなので、迷ったら Temurin（Eclipse Foundation） か Oracle JDK を選べばOKです。

ディストリビューション 提供元 ライセンス おすすめ場面
Eclipse Temurin Eclipse Foundation (Adoptium) 無償 ✅ 最もおすすめ。学習・業務ともに
Oracle JDK Oracle 個人・開発は無償 ✅ Oracle の公式版。企業での商用利用は有償条件あり
Amazon Corretto Amazon 無償 AWS を使う場合に相性が良い
Microsoft Build of OpenJDK Microsoft 無償 Azure 利用時に便利
GraalVM Oracle / GraalVM 無償（CE） 高速起動・ネイティブコンパイルが必要な場合
🪟 Windows へのインストール
🪟 Windows
1
Eclipse Temurin のダウンロード
https://adoptium.net/temurin/releases/ にアクセスします。
フィルターで Version: 21、OS: Windows、Architecture: x64、Package Type: JDK、File Type: .msi を選択してダウンロードします。
2
インストーラを実行する
ダウンロードした .msi ファイルをダブルクリックしてインストーラを起動します。
「Add to PATH」と「Set JAVA_HOME」にチェックが入っていることを確認してから「Next」を進めます。これにより環境変数が自動設定されます。
3
インストールの確認
インストール完了後、コマンドプロンプトまたは PowerShell を新しく開き（既に開いているものは閉じて再起動）、以下のコマンドを実行します。
コマンドプロンプト / PowerShell 
C:\Users\you> java -version
openjdk version "21.0.x" 2024-xx-xx LTS
OpenJDK Runtime Environment Temurin-21.0.x+xx (build 21.0.x+xx-LTS)
OpenJDK 64-Bit Server VM Temurin-21.0.x+xx (build 21.0.x+xx-LTS, mixed mode, sharing)

C:\Users\you> javac -version
javac 21.0.x

✅ 両方バージョンが表示されればインストール成功！
⚠️ 「'java' は認識されていない…」と表示される場合
コマンドプロンプトを一度閉じて再起動してみてください。それでも表示される場合は、環境変数 PATH に JDK の bin フォルダが追加されていない可能性があります。「システムの詳細設定」→「環境変数」→「Path」に C:\Program Files\Eclipse Adoptium\jdk-21\bin を追加してください。
🍎 macOS へのインストール
🍎 macOS

macOS では Homebrew（パッケージマネージャ）を使うのが最も簡単です。
または公式サイトから .pkg インストーラをダウンロードする方法もあります。

方法A：Homebrew を使う（推奨）

1
Homebrew のインストール（まだ入っていない場合）
ターミナルを開き（Spotlight で「Terminal」検索）、以下を実行します。
ターミナル 
# Homebrew のインストール（既に入っている場合はスキップ）
% /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Homebrew の動作確認
% brew --version
Homebrew 4.x.x
2
Eclipse Temurin JDK 21 のインストール
以下のコマンドを実行します。
ターミナル 
# Adoptium の tap を追加（初回のみ）
% brew tap homebrew/cask-versions

# JDK 21 (Temurin) をインストール
% brew install --cask temurin@21

# インストールの確認
% java -version
openjdk version "21.0.x" 2024-xx-xx LTS

% javac -version
javac 21.0.x

✅ 両方バージョンが表示されればインストール成功！

方法B：インストーラ（.pkg）を使う

https://adoptium.net/temurin/releases/ から、OS: macOS・Architecture: aarch64（M1/M2/M3 Mac）または x64（Intel Mac）・File Type: .pkg を選んでダウンロードし、ダブルクリックでインストールします。

💡 Apple Silicon (M1/M2/M3) の場合
Architecture は aarch64 を選んでください。x64 版も動作しますが Rosetta 2 経由になるため、ネイティブの aarch64 版の方が高速です。
🐧 Linux へのインストール
🐧 Linux（Ubuntu / Debian 系）
ターミナル（Ubuntu / Debian） 
# パッケージリストを更新
$ sudo apt update

# OpenJDK 21 をインストール
$ sudo apt install -y openjdk-21-jdk

# インストールの確認
$ java -version
openjdk version "21.0.x" 2024-xx-xx
OpenJDK Runtime Environment (build 21.0.x+xx-Ubuntu-x)

$ javac -version
javac 21.0.x

# JAVA_HOME の確認（任意）
$ echo $JAVA_HOME
# 何も表示されない場合は以下で設定
$ export JAVA_HOME=$(dirname $(dirname $(readlink -f $(which java))))

✅ バージョンが表示されればインストール成功！

複数バージョンの JDK が入っている場合、sudo update-alternatives --config java で使用するバージョンを切り替えられます。

💻 IDE（統合開発環境）の選択

テキストエディタとターミナルだけで Java は書けますが、IDE を使うと補完・デバッグ・リファクタリングが格段に楽になります。

🧠
IntelliJ IDEA
Community（無償）
世界で最もよく使われる Java IDE。補完・リファクタリング・デバッガが非常に優秀。
Community 版は無償で本コースの内容は全てカバーできます。
迷ったらこれを選びましょう。
📝
VS Code
無償
軽量なエディタ。「Extension Pack for Java」プラグインをインストールすることで Java 開発環境になります。
すでに VS Code を使っている方向け。
🌑
Eclipse
無償
長年使われてきた老舗の Java IDE。企業の既存プロジェクトで指定されることがあります。
現在は IntelliJ IDEA に移行する場合が多いです。

IntelliJ IDEA Community のインストール

https://www.jetbrains.com/idea/download/ にアクセスし、Community 版（無償）をダウンロードしてインストールします。起動時に JDK の場所を聞かれるので、先ほどインストールした JDK のパスを指定します。

💡 IntelliJ IDEA は JDK を自動検出します
JDK を正しくインストールしていれば、IntelliJ IDEA は自動的に JDK を見つけてくれます。
新規プロジェクト作成時に「JDK: 21」が表示されれば設定完了です。
👋 最初のプログラム「Hello, World!」を動かす

環境が整ったら、最初のプログラムを書いて実行してみましょう。

① ファイルを作成する：Main.java

任意のフォルダに Main.java という名前でファイルを作成し、以下のコードを書きます。

Main.java 
public class Main {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
⚠️ ファイル名とクラス名は一致させる
public class Main と書いたら、ファイル名は必ず Main.java にしてください（大文字小文字も含めて完全一致）。
② コマンドラインで実行する

Main.java を保存したフォルダに移動して、以下のコマンドを実行します。

ターミナル / コマンドプロンプト 
# Main.java があるフォルダに移動
$ cd /path/to/your/folder

# コンパイル（Main.class が生成される）
$ javac Main.java

# 実行（.class は不要）
$ java Main
Hello, World!

# Java 11 以降は直接実行も可能（単一ファイルの場合）
$ java Main.java
Hello, World!
③ IntelliJ IDEA での実行
  1. IntelliJ IDEA を起動 → 「New Project」 をクリック
  2. 「Java」 を選択し、JDK に 「21」 を指定 → 「Next」
  3. プロジェクト名を入力（例：HelloJava）→ 「Finish」
  4. src フォルダを右クリック → 「New」→「Java Class」→ Main と入力
  5. main メソッドを書いて、左の ▶ ボタンを押すか Shift+F10 で実行
✅ 下部の「Run」ウィンドウに「Hello, World!」が表示されたら成功！
🔀 複数バージョンの JDK を管理する（上級者向け）

複数のプロジェクトで異なる Java バージョンを使い分けたい場合は、バージョン管理ツールが便利です。

SDKMAN!（macOS / Linux）— https://sdkman.io/ 
# SDKMAN! のインストール
$ curl -s "https://get.sdkman.io" | bash
$ source "$HOME/.sdkman/bin/sdkman-init.sh"

# 利用可能な Java 一覧を表示
$ sdk list java

# Temurin 21 をインストール
$ sdk install java 21.0.x-tem

# デフォルトバージョンの切り替え
$ sdk use java 17.0.x-tem
🔧 よくあるトラブルと解決方法
❌ 「'java' は内部コマンドまたは外部コマンドとして認識されません」（Windows）
環境変数 PATH に JDK の bin フォルダが追加されていません。
「Temurin」インストーラの場合：C:\Program Files\Eclipse Adoptium\jdk-21.x.x.xx-hotspot\bin を PATH に追加してください。その後コマンドプロンプトを再起動してください。
❌ 「Error: Could not find or load main class Main」
javac Main.java でコンパイルしてから java Main で実行しているか確認してください。
また クラス名とファイル名が一致しているか（大文字小文字も）確認してください。java Main.class ではなく java Main（.class 不要）です。
❌ 「エラー: ソース・リリースXXには、ターゲット・リリースYYが必要です」
コンパイルに使った JDK と実行に使う JVM のバージョンが異なっています。
java -version と javac -version を両方実行して、同じバージョンを使っているか確認してください。
❌ IntelliJ IDEA で「SDK not configured」と表示される
メニューの「File」→「Project Structure」→「SDKs」で「+」をクリックし、JDK をインストールしたフォルダを指定してください。
Temurin の場合（Windows）：C:\Program Files\Eclipse Adoptium\jdk-21.x.x.xx-hotspot
❌ macOS で「xcrun: error」が出る
Xcode Command Line Tools が必要な場合があります。
xcode-select --install を実行してインストールしてください。
