アクセス修飾子とパッケージの関係

アクセス修飾子はクラス・フィールド・メソッドの「見える範囲」を制御します。
パッケージを意識した設計において、修飾子の選択が重要になります。

private
そのクラス内のみアクセス可能。外部に公開したくないフィールド・実装メソッドに使う。
（なし）
同じパッケージ内のみアクセス可能（package-private）。パッケージ内部の連携処理に使う。
protected
同じパッケージ＋サブクラスからアクセス可能。継承を前提とした設計で使う。
public
どこからでもアクセス可能。外部に公開する API に使う。

アクセス範囲の比較

修飾子 同クラス 同パッケージ サブクラス どこでも
private ✅ ❌ ❌ ❌
（なし） ✅ ✅ ❌ ❌
protected ✅ ✅ ✅ ❌
public ✅ ✅ ✅ ✅

パッケージ設計の原則

com.example/
├─ model/　← ドメインオブジェクト（データ）
│   ├─ User.java
│   └─ Product.java
├─ repository/　← データアクセス層
│   └─ UserRepository.java
├─ service/　← ビジネスロジック層
│   └─ UserService.java
├─ util/　← 汎用ユーティリティ
│   └─ StringUtils.java
└─ Main.java　← エントリポイント
💡 パッケージ命名の慣習
・ドメイン名を逆順にする：com.yourcompany.projectname
・すべて小文字で書く：com.example.model（大文字は使わない）
・機能や役割でサブパッケージを作る：model・service・util など

package-private（修飾子なし）の活用

パッケージ内部でのみ使うクラスやメソッドは修飾子を省略してパッケージ内に閉じます。

パッケージ内部用クラス（修飾子なし）
package com.example.util;

// package-private：同パッケージからしかアクセスできない
class InternalHelper {
    static String normalize(String s) {
        return s.trim().toLowerCase();
    }
}

// public：外部から使えるクラス
public class StringUtils {
    public static boolean isNullOrEmpty(String s) {
        if (s == null) return true;
        return InternalHelper.normalize(s).isEmpty();
    }
}

実践的なパッケージ構成例

シンプルな商品管理システムの構成
// com/example/model/Product.java
package com.example.model;

public class Product {
    private final int    id;
    private       String name;
    private       int    price;

    public Product(int id, String name, int price) {
        this.id    = id;
        this.name  = name;
        this.price = price;
    }

    public int    getId()    { return id; }
    public String getName()  { return name; }
    public int    getPrice() { return price; }
    public void   setPrice(int price) { this.price = price; }

    @Override
    public String toString() {
        return "[" + id + "] " + name + " (" + price + "円)";
    }
}

// com/example/service/ProductService.java
package com.example.service;

import com.example.model.Product;
import java.util.*;
import java.util.stream.*;

public class ProductService {
    private List<Product> products = new ArrayList<>();

    public void add(Product p)    { products.add(p); }

    public List<Product> getAll() { return Collections.unmodifiableList(products); }

    public Optional<Product> findById(int id) {
        return products.stream()
            .filter(p -> p.getId() == id)
            .findFirst();
    }

    public List<Product> findByMaxPrice(int maxPrice) {
        return products.stream()
            .filter(p -> p.getPrice() <= maxPrice)
            .collect(Collectors.toList());
    }
}

// com/example/Main.java
package com.example;

import com.example.model.Product;
import com.example.service.ProductService;

public class Main {
    public static void main(String[] args) {
        ProductService svc = new ProductService();
        svc.add(new Product(1, "りんご",  150));
        svc.add(new Product(2, "バナナ",  120));
        svc.add(new Product(3, "メロン", 1200));

        svc.findByMaxPrice(200).forEach(System.out::println);
    }
}
▶ 出力結果
[1] りんご (150円)
[2] バナナ (120円)

package com.example;

import java.util.*;
import java.util.stream.*;
import static java.lang.Math.*;

public class Q1 {

    // package-private メソッド
    static int clamp(int v, int lo, int hi) {
        return max(lo, min(hi, v));
    }

    public static void main(String[] args) {
        // static import で Math メソッドを使う
        System.out.println(pow(2, 8));
        System.out.println(sqrt(625.0));
        System.out.println(abs(-100));

        // package-private メソッド
        System.out.println(clamp(50, 0, 100));
        System.out.println(clamp(-5, 0, 100));
        System.out.println(clamp(150, 0, 100));

        // Stream + Math
        List<Integer> nums = List.of(-3, 7, -1, 5, -9, 2);
        int sumAbs = nums.stream()
            .mapToInt(n -> abs(n))
            .sum();
        System.out.println("絶対値の合計: " + sumAbs);

        long negCount = nums.stream().filter(n -> n < 0).count();
        System.out.println("負の数の個数: " + negCount);
    }
}

Q2.java
import java.util.*;
import java.util.stream.*;

public class Q2 {

    // 不変データクラス（record）
    record Point(int x, int y) {
        double distanceTo(Point other) {
            int dx = this.x - other.x;
            int dy = this.y - other.y;
            return Math.sqrt(dx * dx + dy * dy);
        }

        @Override
        public String toString() {
            return "(" + x + "," + y + ")";
        }
    }

    public static void main(String[] args) {
        List<Point> points = List.of(
            new Point(0, 0),
            new Point(3, 4),
            new Point(1, 1),
            new Point(6, 8)
        );

        Point origin = new Point(0, 0);

        // 原点からの距離でソート
        points.stream()
            .sorted(Comparator.comparingDouble(
                p -> p.distanceTo(origin)))
            .forEach(p -> System.out.printf(
                "%s 距離:%.1f%n", p, p.distanceTo(origin)));

        System.out.println("---");

        // 距離が5以上のポイント数
        long count = points.stream()
            .filter(p -> p.distanceTo(origin) >= 5)
            .count();
        System.out.println("距離5以上: " + count + "個");

        // toMap で Point → 距離
        Map<String, Double> distMap = points.stream()
            .collect(Collectors.toMap(
                Point::toString,
                p -> p.distanceTo(origin)));
        new TreeMap<>(distMap).forEach(
            (k, v) -> System.out.printf("%s → %.2f%n", k, v));
    }
}

解説
  • 距離：(0,0)=0、(1,1)=√2≈1.41、(3,4)=√25=5.0、(6,8)=√100=10.0。
  • 距離5以上：(3,4)と(6,8) = 2個。
  • TreeMap でキー（文字列）の昇順で出力：(0,0)・(1,1)・(3,4)・(6,8)。
問題 14-3-3（総合問題）
コード記述 ⭐⭐⭐

本コースの総仕上げとして、次の条件を満たすシステムを設計してください。

  • com.example.model パッケージに Student（record）を定義：name・score・grade（学年）フィールド
  • com.example.service パッケージに StudentService を定義：List<Student> を保持し、以下の機能を持つ
    • add(Student)：学生を追加
    • getTopN(int n)：点数上位 N 名を返す
    • getAverageByGrade()：学年別平均点を Map<Integer,Double> で返す
    • getPassedStudents(int threshold)：閾値以上の学生名リストを返す
  • com.example.Main で各機能を呼び出す

com/example/model/Student.java
package com.example.model;

public record Student(String name, int score, int grade) {
    @Override
    public String toString() { return name + "(" + score + ")"; }
}
com/example/service/StudentService.java 
package com.example.service;

import com.example.model.Student;
import java.util.*;
import java.util.stream.*;

public class StudentService {
    private List<Student> students = new ArrayList<>();

    public void add(Student s) { students.add(s); }

    public List<Student> getTopN(int n) {
        return students.stream()
            .sorted(Comparator.comparingInt(Student::score).reversed())
            .limit(n)
            .collect(Collectors.toList());
    }

    public Map<Integer, Double> getAverageByGrade() {
        return students.stream()
            .collect(Collectors.groupingBy(
                Student::grade,
                Collectors.averagingInt(Student::score)));
    }

    public List<String> getPassedStudents(int threshold) {
        return students.stream()
            .filter(s -> s.score() >= threshold)
            .sorted(Comparator.comparing(Student::name))
            .map(Student::name)
            .collect(Collectors.toList());
    }
}
com/example/Main.java
package com.example;

import com.example.model.Student;
import com.example.service.StudentService;
import java.util.*;

public class Main {
    public static void main(String[] args) {
        StudentService svc = new StudentService();
        svc.add(new Student("Alice", 85, 1));
        svc.add(new Student("Bob",   72, 2));
        svc.add(new Student("Carol", 91, 1));
        svc.add(new Student("Dave",  60, 3));
        svc.add(new Student("Eve",   78, 2));
        svc.add(new Student("Frank", 95, 3));

        System.out.println("=== 上位3名 ===");
        svc.getTopN(3).forEach(System.out::println);

        System.out.println("=== 学年別平均 ===");
        new TreeMap<>(svc.getAverageByGrade()).forEach(
            (g, avg) -> System.out.println(g + "学年: " + avg));

        System.out.println("=== 合格者（75点以上） ===");
        System.out.println(svc.getPassedStudents(75));
    }
}
解説
  • パッケージ分離によって model（データ）・service（ビジネスロジック）・エントリポイントが明確に分かれます。
  • record で不変のデータクラスを簡潔に定義します。
  • StudentService は Stream API を活用してソート・グループ化・フィルタを行います。
  • 1学年：(85+91)/2=88.0、2学年：(72+78)/2=75.0、3学年：(60+95)/2=77.5。
